# ü•ê Croissant Stack UI Components

Welcome to the **Croissant Stack** UI component library documentation. This Storybook showcases all the reusable components available in the `@repo/ui` package.

## Getting Started

All components are built with:
- **React 19** - The latest React with improved performance
- **Tailwind CSS v4** - Utility-first CSS framework
- **Radix UI** - Unstyled, accessible component primitives
- **Class Variance Authority** - For managing component variants

## Installation

Components are available from the `@repo/ui` package:

```tsx
import { Button } from "@repo/ui/components/button";
import { Card } from "@repo/ui/components/card";
import { Input } from "@repo/ui/components/input";
```

## Component Categories

### Forms
- **Button** - Clickable button with multiple variants
- **Input** - Text input field
- **Textarea** - Multi-line text input
- **Checkbox** - Toggleable checkbox
- **Switch** - Toggle switch
- **Select** - Dropdown select
- **RadioGroup** - Radio button group
- **Slider** - Range slider
- **Label** - Form labels

### Layout
- **Card** - Container with header, content, footer
- **Separator** - Visual divider
- **Tabs** - Tabbed navigation
- **Accordion** - Collapsible sections

### Feedback
- **Alert** - Alert messages
- **Badge** - Status badges
- **Progress** - Progress indicator
- **Skeleton** - Loading placeholder
- **Tooltip** - Hover tooltips

### Overlay
- **Dialog** - Modal dialogs
- **Sheet** - Slide-out panels
- **Popover** - Floating content
- **DropdownMenu** - Contextual menus

### Data Display
- **Avatar** - User avatars
- **Table** - Data tables

## Theming

The components support both light and dark modes through CSS custom properties. Toggle the theme using the toolbar above.

## Accessibility

All components are built with accessibility in mind, following WAI-ARIA guidelines and keyboard navigation patterns.
